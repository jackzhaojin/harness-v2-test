# Task 15: Team members page with grid and filtering

## Goal
Build the Team page with responsive card grid layout, member cards showing details and online status, and filtering by search and role dropdown. Implements Stories 18 and 19.

## Acceptance Criteria
- [ ] Team page renders at /team route
- [ ] Members displayed in responsive card grid layout
- [ ] Grid: 1 column mobile, 2 columns tablet, 3 columns desktop
- [ ] Each card shows: large avatar, name, role, email, online status indicator
- [ ] Online status: green dot for online, gray dot for offline
- [ ] Email is clickable mailto link
- [ ] Cards have consistent height regardless of content
- [ ] Cards have hover effect (shadow or border change)
- [ ] Search input filters members by name (case-insensitive, real-time)
- [ ] Role dropdown filter with options: All, Developer, Designer, Manager
- [ ] Filters can be combined (search + role together)
- [ ] Empty state message when no members match filters
- [ ] Filter state resets on page navigation
- [ ] Result count shown (e.g., Showing 3 of 8 members)
- [ ] All cards adapt properly to dark mode

## Required Checks
- [ ] Smoke: App loads without errors


## Constraints
- Follow patterns from SPEC/HOW.md
