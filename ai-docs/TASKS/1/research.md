## Technical Approach

### Implementation Strategy

The task involves creating a single HTML file (`index.html`) that serves as the entire application entry point. Following the "Single-file deployability" principle from HOW.md, both CSS and JS will be embedded directly in the HTML file.

**HTML Structure**: The file will use semantic HTML5 elements as specified: `<header>` for the app title, `<main>` for the primary content, `<form>` for the input section with an input field and submit button, and `<ul>` with `<li>` elements for the todo list. ARIA labels will be applied per accessibility requirements.

**CSS Architecture**: The styles will be organized in a logical order within the `<style>` block: CSS custom properties (variables) for theme values, a CSS reset, layout styles, component styles, and state styles. The design will be mobile-first, starting with base styles and using media queries for larger viewports.

**Empty State**: A dedicated empty state container will be included in the initial HTML with a message like "No todos yet". Initially visible, this will later be toggled via JavaScript based on todo count.

**Playwright Infrastructure**: The existing test configuration at `tests/e2e/playwright.config.js` is already set up to test against `../../index.html`.
