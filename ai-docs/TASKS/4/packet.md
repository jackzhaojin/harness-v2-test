# Task 4: Reusable UI component library

## Goal
Build the complete set of reusable UI components including Button, Card, Badge, Input, Select, Toggle, Avatar, ProgressBar, and Dropdown. All components support variants, sizes, dark mode, and accessibility requirements.

## Acceptance Criteria
- [ ] Button component with variants: primary, secondary, outline, ghost
- [ ] Button component with sizes: sm, md, lg and loading state with spinner
- [ ] Button has visible focus ring and disabled state styling
- [ ] Card component with configurable padding and shadow variants
- [ ] Badge component with color variants for status (green, yellow, blue, red, gray)
- [ ] Input component with label, placeholder, error state, and dark mode support
- [ ] Select component with options array, placeholder, and controlled value
- [ ] Toggle component (switch) with on/off states and accessible labeling
- [ ] Avatar component displaying image or initials fallback with size variants
- [ ] ProgressBar component showing percentage with visual bar fill
- [ ] Dropdown component with trigger button and menu items, closes on click outside
- [ ] All components have TypeScript props interfaces
- [ ] All components support dark mode via dark: Tailwind classes
- [ ] All interactive components have focus:ring-2 focus states
- [ ] Components are keyboard accessible (Tab navigation, Enter/Space activation)

## Required Checks
- [ ] Smoke: App loads without errors
