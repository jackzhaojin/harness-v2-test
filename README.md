# harness-v2-test

Monorepo for projects generated by autonomous harness pipelines.

Each project lives in its own directory under `projects/`. Projects are generated by various harness systems (study harness, app builders, etc.) and run independently.
Deployments are published to GitHub Pages under `https://jackzhaojin.github.io/harness-v2-test/projects/<project>/`.

## Structure

```
projects/
├── aem-devops-cert-v1/  ← AEM DevOps certification study environment
├── claude-cert-v1/      ← Claude certification study environment
└── (future projects...)
```

## GitHub Pages

Deployable projects are discovered automatically from `projects/` when they contain:

- a `package.json`
- a Vite config file
- a working `npm run build`

The Pages workflow runs `node scripts/build-pages-site.mjs`, which:

- installs each deployable project's dependencies
- builds each project with a Pages-safe base path
- stages output into `site/projects/<project>/`
- generates a repo-level landing page and SPA fallback

This keeps every project deployable from one repo-site artifact instead of forcing a separate Pages configuration per app.
